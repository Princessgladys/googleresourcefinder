What Address Do I Email?

To update Resource Finder by email, use the email address pattern subdomain-updates@resource-finder.appspotmail.com. For example:

Pakistan: pakistan-updates@resource-finder.appspotmail.com

Haiti: haiti-updates@resource-finder.appspotmail.com

Data Input Types

Each attribute in the system accepts values of a specific kind. The different types of fields are as follows:

Plain text: any value is accepted
Numeric: values must be numbers only
Boolean: values must be ‘yes’ or ‘no’
Geopoint: values must be two floating point numbers, separated by a comma, formatted as latitude north of equator, then longitude west of Greenwich
Checklist: any number of values from a predefined set of plain text options*
Choice: one of a predefined set of plain text options*
Unspecified: if at any point you wish to erase the value for an attribute, the value must be ‘*none’

* Accepted values are restricted to a predefined list of choices. Each attribute may have its own list of possible choices.

Attributes

Each facility type has several attributes that can be updated with this system. When making changes, the attributes may be specified in a case insensitive manner. Using “Available beds” is equivalent to both “Available Beds” and “available beds”, or even “aVAILABLE bEDS”. Example methods of describing an attribute are listed below:

Name: a short name for the field. These names may include any standard character in the roman alphabet [A-Z], upper or lower case.

Examples: Available beds, Services

Abbreviation: an abbreviated name for an individual attribute. Uses the same characters as the name of an attribute.

Examples: avbed, serv

Both versions, the full name or abbreviation, are acceptable as identifiers for an attribute in an update email.

Drafting a New Email Update

To update a single facility, you must first declare which facility is being updated. To do so, begin your email like this:

update General Hospital

where “General Hospital” is the name of the hospital to be edited. Each line after should specify a specific attribute to be updated, a colon, and a value to update the attribute to. For example:

Available beds: 50

As shown, the format for each line should be “Attribute: value for the attribute”. Multiple lines may be put in a row, like so:

Available beds: 50
Total beds: 100

Similarly, abbreviations may be used as mentioned above:

avbed: 50
totbed: 100

Put together, if we wanted to update the hospital “General Hospital” to show that there are 50 available beds and 100 total beds, we might write the following:

update General Hospital
avbed: 50
totbed: 100

If we wanted to update multiple facilities in one email, simply append a similar section for the second hospital below the first, like so:

update General Hospital
avbed: 50

update Public Hospital
avbed: 75

Will I Receive Confirmation?

After your email has been sent, you should receive email containing a list of the updates the server accepted. In addition, you’ll receive a list of any errors you made in your email, as well as a short reference document to remind you of the basics, just in case.

Assuming the email you receive does not explicitly mention any errors, no further action is required for your update to go through on the server.

Using Different Data Input Types

Below you’ll find a description of how to specify each data input type, as well as an example of each.

Plain text

Plain text lines simply follow the format above, where any value may be input after the colon, be it a word, phrase, number, etc.

Contact name: John Smith
Address: 5 Avenue Rd

Numeric

Numeric attributes require the value to be a number.

Total beds: 100

Boolean

Boolean values can be set with a simple ‘yes’ or ‘no’. The values ‘y’, ‘n’, ‘true’, and ‘false’ are also acceptable as alternatives.

Reachable by road: yes
Can pick up patients: false

Checklist

Checklists are made up of a collection of values, chosen from a specific list of accepted values. For example, if the “services” field has the options “Cardiology”, “Pediatrics”, and “General Surgery”, you would select values as such:

Services: Cardiology, Pediatrics

This above example would set the value for “Services” to both “Cardiology” and “Pediatrics”. As you can see, each value is simply separated by a comma. You can also use a plus/minus syntax to add or remove individual choices, rather than overwriting the entire list. To do so, format the value like so:

Services: -Cardiology, +General Surgery

This example would remove “Cardiology” from the list of values, as well as add “General Surgery”. As we also set “Pediatrics” earlier, that value would still remain.

Choice

Multiple choice values are similar to checklists, except you may only select one value instead of many. For example, if dealing with the attribute “Operational status”, which has the values “Open” and “Closed”, we would set it to “Open” like this: 

Operational status: Open

Clearing the Value for an Attribute

Unspecified values are a special case. Any attribute may be cleared with the special keyword “*none”. This action effectively erases the current value. From that point on, accessing the value for an attribute will return nothing until it has been set again to a new value. To do so, use the following syntax:

Services: *none

How to Get Help

If you send email with no updates to the server, you will automatically receive a templated email with instructions in reply. This email will be tailored to the default subject type in the subdomain you have emailed.

If you wish the form to automatically include the title for a specific facility, set the subject line of your email to that title. For example, to update the facility “General Hospital” in the subdomain “Pakistan”, you would enter the following information into your email:

To: pakistan-updates@resource-finder.appspotmail.com
Subject: General Hospital
Body: <blank>

Error Handling

If at any point you make an error in your email update to the server, you will receive email back, pointing out your mistake.

By mistake, a user might send the server an email like this:

update General Hospital
Available beds: fifty

In response, they would receive the following error message:

update General Hospital
Available beds: fifty
-- "Available beds" requires a numerical value.

The third line, indented by the two dashes, is where the error message is located. In this particular case, as the attribute “available beds” is a numerical attribute, typing in a word (even though fifty is a textual representation of the number 50) is not a valid value.

If the attribute in question is a checklist or multiple choice type attribute, then a list of accepted values will follow the error message. For example, an incorrect attempt to update the multiple choice “Services” attribute will result in the following error message:

update 128 Chak Hospital
Services: x
-- "Services" requires all values to be from a specific set.
-- Accepted values are: General surgery, Orthopedics, Neurosurgery,
---- Vascular surgery, Internal medicine, Cardiology

There may be cases when multiple facilities have the same name. Attempting to update a facility in that case will result in an error message asking you to choose the one you wish to update. For example, if there are three facilities with the name “General Hospital”, an attempted update will return a list of facilities, each accompanied by a unique ID:

General Hospital (example.org/123)
General Hospital (example.org/456)
General Hospital (example.org/789)

To find which facility to update, visit the Resource Finder application and click the icon on the map corresponding to the facility. The info window that opens will display the ID of the facility, which can then be matched to one of the above items. Then retry the update, including the ID. For example:

update General Hospital (example.org/123)
Available beds: 15

would update the facility General Hospital, associated with the key “example.org/123”, to show 15 available beds.
